EXEC OVERVIEW TABLE

Feature	            Description	                    Placeholder / Syntax	Notes
================================================================================================================================
Basic command	    Command to run the program	    Exec=/usr/bin/firefox	Required
Arguments	        Pass command-line arguments	    Exec=firefox %u	        %f, %F, %u, %U, %i, %c, %k
Multiple files	    Open several files	            %F or %U	            Expands to list of files/URLs
Single file	        Open a single file	            %f or %u	            Replaced with one file/URL
Icon substitution	Adds icon reference	            %i	                    Resolves to -icon <Icon> if set
Name substitution	Adds translated name	        %c	                    Replaced with Name field
Desktop file path	Path to .desktop file	        %k	                    Replaced with absolute path of the .desktop file
Field escaping	    Handle spaces / special chars	Quotes "..."	        Important on multi-word paths or args
Chaining / shell	Run multiple commands	        sh -c 'cmd1; cmd2'	    Only when needed, beware security
Environment	        Run with env variables	        env VAR=val program	    Optional, sometimes cleaner than shell wrapper
Terminal execution	Run in terminal window	        Terminal=true	        Must match expectations for CLI apps
Shell wildcards	    Pass glob patterns	            *	                    Rarely used in .desktop, mostly %F/%U preferred




DETAILED EXPLANATION OF PLACEHOLDERS

Placeholder	Replaces with	        Use case
================================================================================================================================
%f	        Single filename	        Launch app on one local file
%F	        Multiple filenames	    Batch open files (drag/drop)
%u	        Single URL	            Open a website or resource
%U	        Multiple URLs	        Open multiple web resources
%i	        Icon parameter	        Adds -icon <Icon> if relevant
%c	        Translated Name	        Localized app name
%k	        Desktop file path	    Full .desktop path, used for logging or launching copy

MimeType= declares what kinds of files or data an application can open.
This is how apps become selectable in “Open With…” and how defaults are chosen.

Relationship to Exec= (critical linkage)

If you declare MimeType, your Exec= must support it.

MIME usage	        Required Exec placeholder
================================================================================================================================
Single file	        %f
Multiple files	    %F
URL-based types	    %u / %U





Actions= — macro overview table

Actions= defines additional launcher actions, usually shown in:

Right-click context menus

Long-press menus (touch)

Dock / panel menus (DE-dependent)
